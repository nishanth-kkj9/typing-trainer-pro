# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 7.x     | :white_check_mark: |

## Reporting a Vulnerability

We take the security of Typing Trainer Pro seriously. If you discover a
security vulnerability, please do **not** open a public issue.

Instead, report it privately by emailing the project maintainer at
nishanth-kkj9 (at) users.noreply.github.com, or by opening a
[GitHub Security Advisory](https://github.com/nishanth-kkj9/typing-trainer-pro/security/advisories/new).

You should expect to receive a response within 72 hours. If the issue
is confirmed, we will release a patch as soon as possible, depending
on complexity.

## Responsible Disclosure

We ask that you:

1. Give us reasonable time to investigate and fix the issue before
   disclosing it publicly.
2. Make a good-faith effort to avoid privacy violations, data
   destruction, or service interruption.
3. Do not exploit the vulnerability beyond what is necessary to
   demonstrate the issue.

## Scope

The following are considered out of scope:

- Issues in third-party dependencies — report them to the respective
  maintainers.
- Attacks requiring physical access to the user's machine.
- Social engineering attacks.
